## 2. Access gate (password `753`)

- **This is a soft, client-side gate — not real security.** The code ships to the browser and can be read/bypassed by anyone inspecting source. It restricts casual public access to a preview only. For genuine protection use HTTP auth / server-side auth.
- Mechanism (zero content flash):
  - `src/partials/head.html` — inline blocking script adds `gate-locked` to `<html>` before paint unless `sessionStorage.ox_gate === 'open'`; plus minimal critical CSS so the overlay covers the page even before `main.css` loads.
  - `src/partials/chrome.html` — gate overlay markup (`#ox-gate`) + a **self-contained inline unlock script** (works even if the JS bundle fails). Correct code `753` sets `sessionStorage.ox_gate=open`, removes `gate-locked`, dispatches `ox:unlocked`.
  - `src/css/main.css` — full gate styling (`#ox-gate`, `.ox-gate__*`, shake-on-error).
- Applies to every page automatically because `chrome.html`/`head.html` are shared partials.
